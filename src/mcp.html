Model Context Protocol

TypeTable in the AI tools you already use

TypeTable speaks MCP. Connect Claude Desktop, Cursor, or any MCP‑compatible client with a single API token, and let your agents read and update your tables — your data becomes structured memory any AI can use.

See the setup Create a token

Why connect over MCP

Real data your agents can trust

Real records, not notes

Agents list tables, query rows, and add or edit records against the same validated data you use in chat — not a scratchpad they forget.

Your rules still apply

Types, required fields, and permissions hold whether a person or an agent writes. A bad value is refused the same way.

Works with your tools

Claude Desktop, Cursor, and any MCP‑compatible client connect over Streamable HTTP with a bearer token.

Setup

Connect in three steps

  1. 1

    Create an API token

    In TypeTable, open Settings → API tokens and create one. Copy it right away — it's shown once and starts with ath_.

  2. 2

    Add TypeTable to your client

    Pick your client below, paste the config, and drop your token into the Authorization header.

  3. 3

    Restart and ask

    Restart the client, then ask it to “list my TypeTable tables” or “add a row to …”. It's talking to your data.

.mcp.json 
{
  "mcpServers": {
    "typetable": {
      "type": "http",
      "url": "https://app.typetable.io/api/mcp",
      "headers": {
        "Authorization": "Bearer ath_YOUR_TOKEN_HERE"
      }
    }
  }
}

Add .mcp.json to your project root — or run claude mcp add --transport http typetable https://app.typetable.io/api/mcp --header "Authorization: Bearer ath_…".

claude_desktop_config.json 
{
  "mcpServers": {
    "typetable": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote@latest",
        "https://app.typetable.io/api/mcp",
        "--allow-http",
        "--transport",
        "http-only",
        "--header",
        "Authorization:${TYPETABLE_MCP_TOKEN}"
      ],
      "env": {
        "TYPETABLE_MCP_TOKEN": "Bearer ath_YOUR_TOKEN_HERE"
      }
    }
  }
}

Settings → Developer → Edit Config. Fully quit and reopen Claude Desktop — a reload isn't enough.

.cursor/mcp.json 
{
  "mcpServers": {
    "typetable": {
      "url": "https://app.typetable.io/api/mcp",
      "headers": {
        "Authorization": "Bearer ath_YOUR_TOKEN_HERE"
      }
    }
  }
}

Add to ~/.cursor/mcp.json (user‑wide) or .cursor/mcp.json in your project, then restart Cursor.

Endpoint https://app.typetable.io/api/mcp
Transport Streamable HTTP
Auth Authorization: Bearer ath_…

Any MCP client that speaks Streamable HTTP works — point it at the endpoint and send the Authorization header.

Tokens are created and revoked in Settings → API tokens; each is shown once and stored only as a hash. Self‑hosting TypeTable? Point your client at your own deployment's /api/mcp instead.
